# Commerce Bank Appointment Booking System

A full-stack appointment booking system for Commerce Bank, built with **React** (frontend) and **Spring Boot** (backend). Customers can book appointments at Kansas City area branches, receive confirmation emails, and download PDF confirmations.

## ✨ Features

- 6-step appointment booking wizard with a professional dark-header UI
- 8 real Commerce Bank branches in the Kansas City area
- Distance calculation from any MO/KS ZIP code within 100 miles of Kansas City using the Haversine formula
- Service filtering — branches that do not offer the selected appointment type are visually disabled
- Real-time time slot availability — booked slots are greyed out for all users instantly
- Backend input validation — invalid or missing fields return a `400 Bad Request` error
- Confirmation email sent via Gmail upon booking
- PDF confirmation download generated by iText 7
- Monday–Friday scheduling, up to 14 business days ahead
- React frontend served directly by Spring Boot — single server, single port

## 🚀 Getting Started

### 1. Clone the Repository

Open a terminal and run:

```bash
git clone https://github.com/ucmo-cs/S26_SE3910_Team3/commerce-bank-appointment-booking.git
cd commerce-bank-appointment-booking
```

---

### 2. Set Up the Backend

#### Open in IntelliJ IDEA

1. Open **IntelliJ IDEA**
2. Click **"Open"**
3. Navigate to the `backend/` folder inside the cloned repository and select it
4. Click **"Trust Project"** if prompted
5. Wait for IntelliJ to finish downloading Maven dependencies (watch the progress bar at the bottom — this may take a few minutes the first time)

#### Configure Email (Required)

The app sends confirmation emails via Gmail. You need to provide your own Gmail credentials:

1. Open `backend/src/main/resources/application.properties`
2. Update these two lines with your Gmail address and App Password:

```properties
spring.mail.username=your-gmail@gmail.com
spring.mail.password=yourapppaswordhere
```

> **How to get a Gmail App Password:**
> 1. Go to [myaccount.google.com](https://myaccount.google.com)
> 2. Click **Security** → enable **2-Step Verification** if not already on
> 3. Search for **"App Passwords"** and click it
> 4. Type any app name and click **Create**
> 5. Copy the 16-character password (no spaces) into `application.properties`

> ⚠️ Never commit real credentials to GitHub. Always use placeholder values in the repository and enter real values locally.

#### Add the ZIP Code Database

The distance calculation requires a CSV file of US ZIP codes:

1. Go to **[simplemaps.com/data/us-zips](https://simplemaps.com/data/us-zips)**
2. Download the free **Basic** version
3. Extract and place `uszips.csv` into:
   ```
   backend/src/main/resources/uszips.csv
   ```

When Spring Boot starts successfully you should see:
```
✅ Loaded 510 ZIP codes within 100.0 miles of Kansas City
```

---

### 3. Set Up the Frontend

Open a terminal in the `frontend/` folder:

**Windows (PowerShell):**
```powershell
cd frontend
npm install
```

**Linux:**
```bash
cd frontend
npm install
```

Then create your environment file:

**Windows:**
```powershell
copy .env.example .env
```

**Linux:**
```bash
cp .env.example .env
```

The default `.env` file works as-is for local development:
```
VITE_API_BASE_URL=http://localhost:8080/api
```

---

### 4. Run the Application

You have two options:

---

#### Option A — Integrated (Recommended)

Everything runs through Spring Boot on a single port (`8080`). Best for testing the complete system.

**Step 1:** Build the React frontend:
```bash
# In the frontend/ folder
npm run build
```

**Step 2:** Copy the build output into Spring Boot:

**Windows:**
```powershell
Remove-Item -Recurse -Force backend\src\main\resources\static\*
Copy-Item -Recurse frontend\dist\* backend\src\main\resources\static\
```

**Linux:**
```bash
rm -rf backend/src/main/resources/static/*
cp -r frontend/dist/* backend/src/main/resources/static/
```

**Step 3:** Start Spring Boot in IntelliJ:
- Open `AppointmentBookingApplication.java`
- Click the green **▶ Run** button or press **Shift+F10**
- Wait until you see `Started AppointmentBookingApplication` in the console

**Step 4:** Open the app:
```
http://localhost:8080
```

> ⚠️ Always use `http://` not `https://` — Spring Boot does not have SSL configured and the browser will show a blank page or throw connection errors if you use HTTPS.

---

#### Option B — Split (For Development)

Runs frontend and backend on separate ports. Best when actively editing React code since changes hot-reload instantly without rebuilding.

**Terminal 1 — Start Spring Boot** (in IntelliJ, press Shift+F10)

**Terminal 2 — Start React dev server:**
```bash
# In the frontend/ folder
npm run dev
```

Open: `http://localhost:5173`

## 🌐 API Endpoints

All endpoints are prefixed with `/api/appointments`.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/appointments` | Book a new appointment |
| `POST` | `/api/appointments/send-confirmation` | Send confirmation email |
| `POST` | `/api/appointments/generate-pdf` | Generate PDF confirmation |
| `GET` | `/api/appointments/branches?zipCode=` | Get branches sorted by distance |
| `GET` | `/api/appointments/booked-slots?branchId=&date=` | Get booked time slots for a branch and date |

### Example — Book an Appointment

```http
POST http://localhost:8080/api/appointments
Content-Type: application/json

{
  "appointmentType": "Loan Consultation",
  "branch": {
    "id": "1",
    "name": "Commerce Bank - Main Street",
    "address": "1000 Walnut Street",
    "city": "Kansas City",
    "state": "MO",
    "zip": "64106",
    "phone": "(816) 234-2000",
    "distance": "0.5 miles",
    "availableServices": ["Loan Consultation"]
  },
  "timeSlot": {
    "date": "2026-06-15",
    "time": "10:00 AM",
    "available": true
  },
  "userInfo": {
    "name": "John Doe",
    "email": "john@example.com",
    "phone": "(555) 123-4567"
  }
}
```

**Response `201 Created`:**
```json
{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "confirmationNumber": "BNK-ABC1234",
  "status": "CONFIRMED",
  "createdAt": "2026-05-01T10:30:00"
}
```

**Validation errors return `400 Bad Request`:**
```json
{
  "error": "Bad Request",
  "message": "Name is required"
}
```

### Making Frontend Changes

1. Edit files in `frontend/src/`
2. Run `npm run dev` to preview changes live at `http://localhost:5173`
3. When ready to deploy to Spring Boot:
   ```bash
   npm run build
   # Copy dist/ contents to backend/src/main/resources/static/
   ```

### Making Backend Changes

1. Edit Java files in `backend/src/main/java/`
2. Press **Shift+F10** in IntelliJ to restart Spring Boot

### Viewing the Database

The app uses an **H2 in-memory database**. Data is wiped on every restart.

While Spring Boot is running, go to:
```
http://localhost:8080/h2-console
```

| Field | Value |
|-------|-------|
| JDBC URL | `jdbc:h2:mem:commercebank` |
| Username | `sa` |
| Password | *(leave blank)* |

Useful queries:
```sql
-- View all appointments
SELECT CONFIRMATION_NUMBER, APPOINTMENT_TYPE, SLOT_DATE, SLOT_TIME,
       BRANCH_NAME, USER_NAME, USER_EMAIL, STATUS FROM APPOINTMENTS;

-- View booked slots for a specific branch and date
SELECT SLOT_TIME FROM APPOINTMENTS
WHERE BRANCH_ID = '1' AND SLOT_DATE = '2026-06-15';
```

## 🧪 Testing with Postman

You can test the API directly using [Postman](https://www.postman.com/downloads/) without going through the UI.

### Setup

1. Set method to `POST`
2. Set URL to `http://localhost:8080/api/appointments`
3. Under **Body** → select **raw** → set format to **JSON**
4. Paste the example request body from the [API Endpoints](#api-endpoints) section above

### Key things to know

- When testing one field, keep all other fields valid so you can isolate which input caused the result
- The backend validates all required fields and returns `400 Bad Request` for invalid input
- To test double-booking: successfully book a slot first, then send an identical request — the second should return `409 Conflict`
- Use `"name": ""` for empty string tests — `"name": ,` is invalid JSON and Postman will reject it before sending

### Valid base values for Postman testing

| Field | Valid value |
|-------|-------------|
| `appointmentType` | `"Loan Consultation"` |
| `branch.id` | `"1"` through `"8"` |
| `timeSlot.date` | Any future weekday e.g. `"2026-06-15"` |
| `timeSlot.time` | `"9:00 AM"` through `"4:00 PM"` (hourly slots) |
| `userInfo.name` | Any non-empty string e.g. `"John Doe"` |
| `userInfo.email` | Valid format e.g. `"john@example.com"` |
| `userInfo.phone` | Any non-empty string e.g. `"(555) 123-4567"` |

## 🐛 Troubleshooting

### "Site can't be reached" at localhost:8080
- Make sure Spring Boot is running in IntelliJ
- Check the IntelliJ console for error messages
- Verify port 8080 is not in use by another application

### Blank page at localhost:8080
- Make sure you copied the `dist/` contents into `backend/src/main/resources/static/`
- Verify `index.html` is directly inside `static/`, not in a subfolder
- Restart Spring Boot after copying

### Browser shows garbled text / SSL errors in console
- You are visiting `https://localhost:8080` — change to `http://localhost:8080`
- Some browsers auto-upgrade to HTTPS; open an incognito window if needed

### ZIP codes not loading / distances show "~5-10 miles"
- Make sure `uszips.csv` is placed in `backend/src/main/resources/`
- Check the IntelliJ console for `✅ Loaded 510 ZIP codes` on startup

### Email not sending
- Double-check your Gmail address and App Password in `application.properties`
- The App Password must have no spaces (remove any spaces Google shows)
- Verify 2-Step Verification is enabled on your Google account
- Check the IntelliJ console for lines starting with `❌`

### Branches not loading
- Make sure Spring Boot is running
- Check the browser console (F12) for error messages
- Test directly: `http://localhost:8080/api/appointments/branches?zipCode=64105`

### Time slots not blocking correctly
- Booked slots reset when Spring Boot restarts (H2 in-memory limitation)
- Do not restart Spring Boot between bookings when testing availability

### API returns 400 for valid-looking input
- Check the response body for the specific validation message
- Make sure all required fields are present and non-empty
- Verify email follows the format `username@domain.extension`

### npm install fails
- Make sure Node.js 18 or higher is installed: `node -v`
- Delete `node_modules/` and `package-lock.json` then run `npm install` again

### Java version errors in IntelliJ
- Make sure Java 17 or higher is installed: `java -version`
- In IntelliJ go to **File → Project Structure → SDK** and set it to Java 17+
